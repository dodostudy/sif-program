- **문제**: 좌측 상단 햄버거 메뉴 버튼이 페이지 제목을 가림
- **수정**: 메뉴 버튼 위치를 제목과 겹치지 않도록 조정 (제목 영역에 좌측 패딩 추가 또는 메뉴 버튼 레이아웃 변경)
- **대상 파일**: `js/components.js`, `css/style.css`

### 1.2 사이드바 헤더 변경 (로고 + 조직명 추가)
- **문제**: 사이드바 상단에 "SIF 위험성평가" 제목만 있고, 소속 조직 정보와 심볼이 없음
- **수정**:
  - "SIF 위험성평가" 제목 왼쪽에 행복플러스+ 분당 심볼 이미지 (`images/logo.png`) 배치
  - 제목 아래에 "분당발전본부 안전관리실" 텍스트 추가
  - 기존 부제 "건설재해 분석 시스템"은 유지
- **대상 파일**: `js/components.js` (renderSidebar 함수)

### 1.3 원형 차트 수치 표시
- **문제**: 도넛/파이 차트에 라벨만 표시되고 건수/비율 수치 없음
- **수정**: Chart.js datalabels 플러그인 활용하여 차트 내부 또는 외부에 `건수 (비율%)` 형태로 표시

### 1.4 용어 변경: "KOEN 공정" → "발전소 공정"
- **문제**: "KOEN"이라는 표현이 특정 기관을 직접 지칭하여 부적절
- **수정**: 모든 페이지에서 "KOEN 공정" → "발전소 공정", "KOEN만" → "발전소 공정만", "KOEN 관련" → "발전소 공정 관련"으로 대체
- **대상 파일**: `js/components.js` (createKoenToggle), 각 HTML 페이지, `CLAUDE.md`

### 1.5 사이드바 메뉴 탭 순서 변경
- **현재 순서**: 대시보드 → 공정조회 → 기인물조회 → 통계분석 → 위험성평가
- **변경 순서**: 대시보드 → 공정조회 → 기인물조회 → **재해형태 분석** → 위험성평가 → 통계분석
- **이유**: 현장 업무 흐름에 맞게 배치 — 공정/기인물/재해형태로 현황 파악 → 위험성평가 수행 → 통계로 종합 분석
- **대상 파일**: `js/components.js` (renderSidebar 메뉴 항목 순서)

## 2. 대시보드 (index.html)

### 2.1 "발전소 공정 관련" 요약 카드 제거
- **현상**: 대시보드 상단 요약 카드 4개 중 "KOEN 관련" (2,530건) 카드가 불필요
- **수정**: 해당 카드 제거. 상단 요약 카드를 3개로 변경 (총 재해건수, 최다 공종, 최다 재해형태)
  - 3개 카드 레이아웃을 `grid-cols-3`으로 조정

### 2.2 공종별 차트 드릴다운 기능 (핵심 신규 기능)
- **요구**: 공종 바 클릭 → 작업명 차트 → 단위작업명 차트로 이어지는 **계층 드릴다운**
- **동작 시나리오**:
  1. 공종별 재해건수 바 차트에서 "마감공사" 클릭
  2. 아래에 마감공사의 **작업명별 차트** 표시 (예: 4.8 판넬 등 외부마감 255건 등)
  3. 작업명 바 클릭 → 해당 작업명의 **단위작업명별 차트** 표시
  4. 각 단계 옆에 연관된 **재해형태 차트** + **기인물 차트** 동시 표시
  5. 바 클릭 시 하단에 **상세 정보 테이블** 표시: 기인물, 재해형태, 재해개요, 재해유발요인, 위험성감소대책 (한 세트)
- **UI 구조**:
  ```
  [공종별 재해건수 바차트]        ← 기존
      ↓ 클릭
  [작업명별 바차트] [재해형태 차트] [기인물 차트]   ← 신규 드릴다운 영역
      ↓ 클릭
  [단위작업명별 바차트] [재해형태 차트] [기인물 차트]
      ↓ 클릭
  [상세 테이블: 기인물 | 재해형태 | 재해개요 | 재해유발요인 | 위험성감소대책]
  ```
- **브레드크럼**: 드릴다운 경로 표시 (예: 마감공사 > 4.8 판넬등 외부마감 > 4.8.2 시공)
- **대상 파일**: `index.html` (대폭 수정 — 드릴다운 영역 추가, 차트 클릭 이벤트, 데이터 필터링)

## 3. 공정조회 (pages/process-inquiry.html)

### 3.1 발전소 공정 선택 시 공종 필터링
- **문제**: 발전소 공정 필터 선택 시 터널공사, 하천 및 항만공사, 교량공사가 공종 드롭다운에 여전히 표시됨 (해당 공종은 발전소 공정 데이터 0건)
- **수정**: 발전소 공정 필터 ON 시, 실제 데이터가 존재하는 공종만 드롭다운에 표시
  - `db.json`에서 `KOEN공정 === true`인 레코드의 공종 unique 값만 추출하여 드롭다운 옵션 갱신
- **대상 파일**: `pages/process-inquiry.html` (setupFilters 함수)

### 3.2 기인물 드롭다운 범위 제한
- **문제**: 공종/작업명/단위작업명을 선택해도 기인물 드롭다운은 전체 58개 항목 그대로 표시
- **수정**: 상위 필터 선택에 따라 해당 공정에서 실제 발생한 기인물만 드롭다운에 표시
  - 예: "마감공사 > 4.8 판넬등 외부마감" 선택 시 → 해당 작업에서 발생한 기인물만 표시
  - `filterDB()`로 현재 필터 조건에 맞는 레코드 추출 → 기인물 unique 값으로 드롭다운 갱신
- **대상 파일**: `pages/process-inquiry.html` (FM subscribe 로직)

### 3.3 재해사례 테이블 내용 전체 표시
- **문제**: 재해개요, 재해유발요인, 위험성감소대책 컬럼이 `line-clamp-2`로 2줄까지만 표시되고 `...`으로 생략
- **수정**: `line-clamp-2` 제거, 전체 내용 표시. 셀 높이 자동 확장
  - `maxWidth` 제한은 유지하되, `wrap: true`와 함께 텍스트 전체 표시
  - 테이블 행 높이가 커지므로 페이지당 표시 건수를 10건으로 조정
- **대상 파일**: `pages/process-inquiry.html` (renderTable 함수의 columns format), `js/utils.js` (buildTable 함수)

## 5. 위험성평가 (pages/risk-assessment.html)

### 5.1 단계별 즉시 결과 표시
- **문제**: 현재 단위작업명(3단계)까지 선택해야만 결과가 표시됨
- **수정**: 각 단계 선택 즉시 해당 범위의 위험성평가 결과를 표시
  - 공종만 선택 → 해당 공종 전체 위험성평가 결과
  - 작업명까지 선택 → 해당 작업명 범위 위험성평가 결과
  - 단위작업명 선택 → 가장 상세한 결과
- **UI 변경**: 위자드 카드를 선택하면 하단에 즉시 결과 표시 (카드 선택 + 결과가 동시에 보임)
- **대상 파일**: `pages/risk-assessment.html` (showResults를 각 단계 selectXX 함수에서 호출)

### 5.2 주요 위험요인 10개로 확대
- **문제**: 현재 Top 5까지만 표시
- **수정**: Top 10으로 확대. 그리드 레이아웃을 `xl:grid-cols-5`로 유지하여 2행 표시
- **대상 파일**: `pages/risk-assessment.html` (renderRiskCards 함수 `.slice(0, 5)` → `.slice(0, 10)`)

### 5.3 재해형태 분포 원형차트 수치 표시
- **문제**: 원형차트에 라벨만 있고 수치 없음
- **수정**: datalabels 플러그인으로 `건수 (비율%)` 표시 (공통 수정사항 1.2와 동일)

### 5.4 재해사례 + 감소대책 한 세트 표시
- **문제**: 현재 재해사례와 감소대책이 별도 섹션으로 분리되어 있음
- **수정**: 기인물별 그룹 내에서 한 세트로 통합 표시
  - 구조: `[기인물명] → 사례1: 재해개요 + 유발요인 + 감소대책 → 사례2: ... → ...`
  - 기존 색상 구분 유지 (사례: 파란계열, 감소대책: 녹색계열)
  - 사례가 여러 개일 경우 **모든 사례의 감소대책을 각각 표시** (현재처럼 중복 제거하지 않음)
- **대상 파일**: `pages/risk-assessment.html` (renderCaseList + renderMitigationList → 통합 함수로 변경)

### 5.5 전체 사례 표시 (생략 없음)
- **문제**: 기인물당 사례 3건만 표시하고 "외 N건" 생략
- **수정**: 모든 사례 표시 (`.slice(0, 3)` 제거). 다양한 감소대책을 볼 수 있도록 전체 펼침
- **대상 파일**: `pages/risk-assessment.html` (renderCaseList 함수)

## 6. 재해형태 분석 (신규 페이지: pages/disaster-type.html)

### 6.0 개요
- **목적**: 재해형태(떨어짐, 부딪힘, 끼임 등)를 기준으로 역방향 분석하여, 해당 재해형태가 어떤 공정·기인물에서 다빈도로 발생하는지 현황을 파악하고 예방대책 수립을 지원
- **활용 시나리오**: "떨어짐 재해 예방계획을 수립하려면 → 떨어짐이 가장 많이 발생하는 공정은? 기인물은? 작업명은? → 해당 사례와 감소대책은?" 순서로 분석
- **대상 데이터**: `data/db.json` (2,574건) — 재해형태 필드 기준 역방향 조회
- **사이드바**: 기존 메뉴에 "재해형태 분석" 항목 추가

### 6.1 필터 구성
- **재해형태 드롭다운**: 전체 20개 재해형태를 건수 내림차순 정렬, `재해형태명 (N건)` 형태 표시
  - 예: `떨어짐 (1,609건)`, `부딪힘 (213건)`, `끼임 (172건)`, ...
- **발전소 공정 필터**: 토글 스위치 (발전소 공정 해당 데이터만 필터링)
- **공종 드롭다운**: 선택한 재해형태에서 실제 데이터가 존재하는 공종만 표시 (건수 내림차순)
- **필터 동작**: 재해형태 선택 → 공종 드롭다운 갱신 → 하위 차트/테이블 전체 갱신
- **초기 상태**: 재해형태 미선택 시 전체 재해형태 요약 현황 표시

### 6.2 요약 현황 (재해형태 선택 전)
- **상단 요약 카드**: 총 재해건수, 재해형태 수, 최다 재해형태(건수/비율)
- **재해형태별 건수 바 차트**: 전체 재해형태를 건수 내림차순 수평 바 차트로 표시
  - 바 클릭 시 해당 재해형태 자동 선택 (필터 연동)
  - datalabels로 건수 + 비율 표시

### 6.3 재해형태 선택 시 상세 분석 (핵심 기능)
- **선택된 재해형태 요약 카드**: 해당 재해형태 총 건수, 전체 대비 비율, 관련 공종 수, 관련 기인물 수

- **공종별 분포 차트** (수평 바 차트)
  - 선택한 재해형태가 발생한 공종별 건수 내림차순
  - datalabels: 건수 (비율%)
  - 예: 떨어짐 선택 → 마감공사 512건(31.8%), 철골공사 298건(18.5%), ...

- **기인물별 분포 차트** (수평 바 차트, Top 15)
  - 선택한 재해형태의 기인물별 건수 내림차순 상위 15개
  - datalabels: 건수 (비율%)
  - 예: 떨어짐 선택 → 비계 125건, 지붕(처마,홈통) 120건, ...

- **작업명별 분포 차트** (수평 바 차트, Top 15)
  - 선택한 재해형태의 작업명별 건수 내림차순 상위 15개
  - 공종 필터 선택 시 해당 공종 내 작업명만 표시

- **재해형태 × 공종 교차 히트맵** (선택 사항)
  - 선택한 재해형태 내에서 공종 × 기인물분류 교차 히트맵
  - 어떤 공종의 어떤 기인물분류에서 해당 재해가 집중되는지 시각화

### 6.4 재해사례 및 예방대책 테이블

## 7. 통계분석 강화 (pages/statistics.html)

### 7.0 개요
- **현재 상태**: 5개 탭 (공종별, 작업명별, 단위작업명별, 기인물별, 교차분석)
- **문제**: 교차분석 탭에 공종×기인물, 기인물×재해형태 히트맵만 존재. 재해형태 기준 분석과 3축(공정·기인물·재해형태) 간 상호 연계 분석이 부족
- **목적**: 기인물↔재해형태↔공정 간 양방향 교차분석을 추가하여, 어떤 조합에서 재해가 집중되는지 종합적으로 파악

### 7.1 기인물별 탭 강화 (기존 탭 확장)
- **추가 차트 1**: 기인물별 주요 재해형태 분포 (스택 바 차트)
  - 상위 15개 기인물 × 재해형태 비중을 스택 바로 표시
  - 예: 비계 → 떨어짐 85%, 부딪힘 10%, 기타 5%
- **추가 차트 2**: 기인물별 주요 공종 분포 (스택 바 차트)
  - 상위 15개 기인물이 어떤 공종에서 주로 발생하는지
  - 예: 비계 → 마감공사 40%, 철골공사 30%, ...

### 7.2 재해형태별 탭 (신규 탭 추가)
- **탭 위치**: 기존 "기인물별" 탭 다음, "교차분석" 탭 앞에 삽입
- **탭명**: "재해형태별"
- **차트 1**: 재해형태별 건수 바 차트 (수평, 내림차순, datalabels)
- **차트 2**: 재해형태별 주요 기인물 분포 (스택 바 차트, Top 10 재해형태)
  - 각 재해형태에서 가장 많이 관여하는 기인물 상위 5개를 스택으로 표시
  - 예: 떨어짐 → 비계 8%, 지붕 7%, 슬라브단부 6%, ...
- **차트 3**: 재해형태별 주요 공종 분포 (스택 바 차트, Top 10 재해형태)
  - 각 재해형태가 어떤 공종에서 주로 발생하는지
  - 예: 떨어짐 → 마감공사 32%, 철골공사 19%, ...
- **차트↔테이블 토글**: 기존 탭과 동일한 UX (차트/테이블 전환 버튼)

### 7.3 공종별 탭 강화 (기존 탭 확장)
- **추가 차트 1**: 공종별 주요 기인물 분포 (스택 바 차트)
  - 12개 공종 × 기인물 상위 5개 스택 표시
  - 예: 마감공사 → 지붕(채광판) 18%, 외부마감재 12%, ...
- **추가 차트 2**: 공종별 주요 재해형태 분포 (스택 바 차트)
  - 12개 공종 × 재해형태 비중 스택 표시
  - 예: 마감공사 → 떨어짐 68%, 부딪힘 12%, ...

### 7.4 교차분석 탭 강화 (기존 탭 확장)
- **기존 유지**: 공종×기인물 히트맵, 기인물×재해형태 히트맵
- **추가 히트맵 1**: 공종 × 재해형태 교차 히트맵
  - 행: 12개 공종, 열: 주요 재해형태
  - 어떤 공종에서 어떤 재해형태가 집중되는지 시각화
- **추가 히트맵 2**: 재해형태 × 공종 × 기인물 Top 조합 테이블
  - 3축 조합 건수 상위 20개를 테이블로 표시
  - 컬럼: 순위, 재해형태, 공종, 기인물, 건수, 비율
  - 예: 1위 — 떨어짐 × 마감공사 × 지붕(채광판) 89건

### 7.5 모바일 반응형
- 스택 바 차트: 모바일에서 1열 풀폭, 범례 하단 배치
- 히트맵: 가로 스크롤 + 폰트 축소
- 3축 조합 테이블: 가로 스크롤 적용

### 7.6 구현 세부사항
- **수정 파일**: `pages/statistics.html` (탭 추가 + 기존 탭 차트 추가)
- **탭 최종 순서**: 공종별 → 작업명별 → 단위작업명별 → 기인물별 → **재해형태별** → 교차분석
- **JS 모듈 활용**: 기존 `data-loader.js`, `chart-manager.js`, `utils.js` 재사용
